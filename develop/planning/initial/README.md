# Initial Planning

This directory is an informal staging area for clarifying the scope, form, objectives, and intended shape of `class-align` before the repository is fully onboarded.

The purpose is to create room for several rounds of thinking rather than compressing all context into a single setup prompt. Notes here may be incomplete, overlapping, exploratory, or occasionally contradictory while the idea develops.

## What belongs here

- Statements about what the repository is for
- Desired outcomes and objectives
- The intended form and overall shape of the project
- Boundaries, constraints, and non-goals
- Open questions, alternatives, and unresolved tensions
- Examples that help communicate intent
- Decisions that become clearer over time

## What does not belong here yet

- Detailed implementation plans
- Prescriptive instructions for how another model must build the project
- Premature architecture or tooling decisions
- Polished canonical documentation presented as settled

## Handoff intent

Once the notes provide enough context, a repository-onboarding model such as Fable or Codex can synthesize them into the initial repository structure and canonical project documentation.

The handoff should communicate the problem, objectives, constraints, and desired shape clearly while leaving the implementation method to the model performing the onboarding.

## Working convention

Add separate Markdown files as new ideas develop. A simple numbered sequence such as `001-scope.md`, `002-objectives.md`, or `003-open-questions.md` can preserve the order of thought without forcing a rigid taxonomy.
